Table 11 provides the row-level branch-unit reliability pass matrix for the audit cards. It translates the source branch-unit summary into descriptive labels while preserving the source and receiver counts, branch counts, source-adequacy statistic, diversity-contrast statistic, and branch-unit decision for each evaluated branch-resolution layer.

A pass indicates that the predefined branch-unit criterion is met for that layer. A fail indicates that the criterion is not met, most often because source adequacy is not above one or the diversity contrast is not negative. A no-call indicates that the row is not scoreable under the predefined branch-unit scoreability rules. Additional sensitivity evidence is reported where available in the source summary; rows without such fields are marked as not reported in the source summary row.

The matrix supports claim tiering under the reliability-aware source-receiver model-checking framework. It should not be interpreted as a global origin classifier, evidence of exact geographic origin, ethnic identity, population continuity, or migration route. For the PH908/R1a application, source-receiver call fields refer to the specified Eastern Europe/Russia-to-Balkans source-receiver test geometry, not to a historical event label or general ancestry assignment.
